@EnableAutoConfiguration
attempts to automatically configure your Spring application based on the jar dependencies that you have added.
Example: if HSQLDB is on your classpath, and you have not manually configured any database connection beans, then Spring Boot auto-configures an in-memory database.
Should be applied to @Configuration class.
@SpringBootApplication
It is @SpringBootConfiguration, @EnableAutoConfiguration and @ComponentScan(excludeFilters={...})
@SpringBootConfiguration
It is a stereotype of @Configuration

Spring Security in web applications (turned on by default in SB):
@EnableWebSecurity for MVC
@Bean public SecurityFilterChain securityFilterChain(HttpSecurity http)
@EnableWebFluxSecurity for reactive model
@Bean public SecurityWebFilterChain securityFilterChain(ServerHttpSecurity http)
Several security configurations can be set in the same time.
Can be separated using @Profile
or using securityMatcher(..) and in case of non @RestController mappings (Custom servlets, H2-Console etc) - use AntPathRequestMatcher in addition.
Also use @Order(1) for method ordering.
Lambda DSL allows to get rid of an and() concatenation method, when building SecurityFilterChain from HttpSecurity.

public AdyenClient() { // constructor
log.info("Environment: " + environment); // null value!
}
environment var is not injected simply because the injection happens after the instance is created using constructor.
Working variant:
public AdyenClient(@Value("${config.environment:ERROR}") String environment) {}
CORS enabling:
The @CrossOrigin annotation enables cross-origin requests on annotated controller methods
@CrossOrigin is supported at the class level, too 
@CrossOrigin(origins = "https://domain2.com", maxAge = 3600)
@RestController
@RequestMapping("/account")
public class AccountController
maxAge - cache of preflight-request OPTIONS. Browser will decide exact time, no longer than maxAge
